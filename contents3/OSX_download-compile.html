Compiling VPython on Mac OSX running X11

Important: This is a limited, out of date version and is no longer supported.
Go to vpython.org

You need to be the "root" (administrative) user to install the software, but because of the dangers of operating as root, Apple hid the root account in OS X. To enable it, boot off the install CD (by holding down the "C" key), and look under the Install menu for the "Password Reset" option. You can use this to change your own password if you forget it, and to enable the root account.

1) Make sure you have the latest update to OSX (OSX 10.2.6 or later) and the latest Apple developer tools (Dec. 2002 or later), available from Apple.

To obtain the Dec. 2002 developer tools, you must register (at no cost) as an Apple developer.
Go to https://connect.apple.com and register.
After successfully logging in, click "Download Software", then click "Developer Tools".
Download "December 2002 Mac OS X Developer Tools".

2) Install X11, available from http://www.apple.com/macosx/features/x11/download

3) Install fink (a program used to install Unix-based OSX applications):

Follow the instructions you find at the fink download site, plus the following comments:
Note that you have to execute gunzip in a terminal emulator on the download file.
After you mount the disk image and doubleclick the installer, you need to
"Click the lock to make changes" in order to get administrator access.
You may have to execute rehash in a terminal emulator to be able to run deselect.
In the dselect program, just press U to update, then Q to quit; don't install anything.

4) Install Python 2.2:
In a "terminal emulator" or "typescript", type these commands:
su root (and give password; need root access to install software)
apt-get install pkgconfig (configuration package needed by VPython installer)
apt-get install gtk+ (graphics toolkit needed by VPython)
apt-get install gtkglarea (more graphics needed by VPython)
apt-get install python22 (Python 2.2 for Mac OSX)
rehash (make sure the system knows about the recent installations)
which python2.2 (you should see /sw/bin/python2.2)
   If not, your search path is not giving preference to /sw/bin, and installation won't work properly.
python2.2 (you should see "Python 2.2"; type 2+2, press Enter, and see 4)
Ctrl-D to exit from Python

5) Install Numeric:
       If you have not already installed the Numeric module for Python (Numpy):

cd /sw/bin (change to directory where Python is located)
ln python2.2 python (make link so that "python" is equivalent to "python2.2")
Go to http://www.pfdubois.com/numpy/ or directly to the Numerical Python Project Page.
Click numpy download and download the platform-independent version Numeric-23.1.tar.gz.
tar -zxf Numeric-23.1.tar.gz (uncompress and extract files; produces Numeric directory)
After unpacking, cd into the Numeric-23.1 folder and execute python setup.py install.

6) Install Visual and other needed components:
Download VPython installer visual-2.1.9-20031005.tar.gz
Edit your environment file ~/.tcshrc to include the following:
        setenv CPPFLAGS -I/sw/include (to reference Fink-installed libraries)
        setenv BROWSER open (to use Safari to read the on-line help)
You can check these are set with "echo $CPPFLAGS" and "echo $BROWSER".
 
Make sure that you are NOT using Apple's recent gcc compiler version 3.3, which has major problems.
g++ --version should show 3.1, not 3.3.
    If it says 3.3, execute gcc_select 3 to change to using version 3.1.

tar -zxf visual-2.1.9-date.tar (uncompress and extract files; produces visual-2.1.9 directory)
cd visual-2.1.9 (enter the Visual directory)
./configure --prefix=/sw
make (compiles the Visual module)
   This will take many minutes and generate lots of output on the screen.
   You may see some warning messages, which normally you can ignore.
su root (and give password; need root access to install software)
make install (installs all components)
The following components were installed into /sw/lib/python2.2/site-packages/visual:
   visual (the *.py components of Visual)
   3D demo programs
   IDLE for VPython (convenient edit/run environment for writing and testing programs)
   documentation for Visual
   A convenience shell script vpython was installed in /sw/bin/vpython.
 
7) Running VPython:
To run VPython, start up X11, and in a typescript execute this:
vpython (starts the IDLE for VPython program editor, starting in the VPython demos directory)
Open demo file bounce2.py in demos directory, press F5.
   You should see a ball bouncing around in a box.
   Right button rotates the camera; middle (left+right) button zooms.

Updating one or more components of VPython:

After installing Python 2.2 and VPython, the following individual component zip files can be downloaded and unzipped. These components are included with the complete VPython package but can be updated without reinstalling the whole package.

Demos, Documentation, Visual module, IDLE for any platform
